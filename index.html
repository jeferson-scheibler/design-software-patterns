Demonstração de Design Patterns

Uma aplicação simples para visualizar os padrões Singleton, Factory Method e Observer em ação.

1. Singleton

Objetivo: Garantir que uma classe tenha apenas uma única instância e fornecer um ponto de acesso global para ela.

Onde está o padrão? A classe `ConfigManager` só pode ser instanciada uma vez. Clicar no botão várias vezes sempre retornará a mesma instância, mantendo o estado (como o tema carregado).

2. Factory Method

                

                    Objetivo: Definir uma interface para criar um objeto, mas deixar as subclasses decidirem qual classe instanciar.
                

                

                    Onde está o padrão? A `NotificationFactory` cria diferentes tipos de notificações (`Email`, `SMS`). O código cliente não precisa saber como cada notificação é criada, apenas solicita o tipo que deseja.

3. Observer

                

                    Objetivo: Definir uma dependência um-para-muitos, de modo que, quando um objeto muda de estado, todos os seus dependentes são notificados e atualizados automaticamente.
                

                

                    Onde está o padrão? A `Newsletter` é o "Subject" (observado). Os `Assinantes` são os "Observers". Quando uma nova notícia é publicada na `Newsletter`, todos os assinantes são notificados e atualizam seu estado automaticamente.
