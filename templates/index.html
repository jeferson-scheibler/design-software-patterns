Demonstração de Design Patterns

Frontend interagindo com um backend Python que implementa os padrões.

1. Builder

Analogia: Montar um Computador peça por peça. O padrão Builder guia a construção de um objeto complexo passo a passo, separando o processo de montagem do resultado final.

Na prática: Clicar nos botões aciona um "Diretor" no backend que usa o `Builder` apropriado (Gamer ou Escritório) para montar o objeto `Computer` com as peças corretas, de forma organizada. 


                    

                

            


            
            

                
2. Prototype

                

                    Analogia: Usar uma Máquina de Clonagem. Em vez de criar um objeto complexo do zero toda vez, você cria uma cópia (clone) de um objeto "protótipo" já existente.
                

                

                    Na prática: O backend tem protótipos de personagens pré-configurados. Clicar no botão aciona o método `clone()` do protótipo escolhido, criando uma nova instância idêntica, mas independente. Note que o ID do clone é diferente do ID do protótipo.
                

                 

                    

                        
                        
                    

                    

                        

                    

                

            


            
            

                
3. Observer

                

                    Analogia: Inscrever-se num Canal do YouTube. Quando o criador (Subject) posta um vídeo novo, todos os inscritos (Observers) são notificados automaticamente.
                

                

                    Na prática: A `NewsAgency` é o Subject. Os canais de TV são os Observers. Quando publicamos uma notícia, a agência avisa todos os canais inscritos, que atualizam as suas manchetes.
