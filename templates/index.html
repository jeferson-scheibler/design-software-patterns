Demonstração de Design Patterns

Frontend interagindo com um backend Python que implementa os padrões.

1. Singleton

Analogia: Pense no Gerente Geral de uma empresa. Só existe um, e todos os departamentos falam com a mesma pessoa para obter as diretrizes oficiais.

Na prática: Clicar no botão sempre vai contatar a mesma instância do `ConfigurationManager` no backend Python. Note que o "ID da Instância" nunca muda. 


                    

                

            


            

                
2. Factory Method

                

                    Analogia: Uma Montadora de Veículos. Você pede um "carro" ou uma "moto", e a fábrica te entrega o produto pronto, sem que você precise saber os detalhes da linha de montagem de cada um.
                

                

                    Na prática: O backend tem uma função `get_exporter`. Pedimos "pdf" ou "csv" e ela nos devolve o objeto certo, pronto para usar, sem que o nosso código precise saber como cada um foi criado.
                

                 

                    

                        
                        
                    

                

            


            

                
3. Observer

                

                    Analogia: Inscrever-se em um Canal do YouTube. Quando o criador (Subject) posta um vídeo novo, todos os inscritos (Observers) são notificados automaticamente.
                

                

                    Na prática: A `NewsAgency` é o Subject. Os canais de TV são os Observers. Quando publicamos uma notícia, a agência avisa todos os canais inscritos, que atualizam suas manchetes.
